## 支持大数据集的隐私计算

实现原理是将大数据集切割成多方子数据集后，然后依次加载子数据集进行计算

子数据集默认切割大小为100w条数据，可通过配置文件中的fc.big_dataset_split_size更改

### BigDataset类型

该类型不存储实际的数据，具体数据存储在bm的fc_cache文件夹中，该类型暂不提供创建函数，而是由datacall库的callWithBigDataset函数返回，当子模型中返回的是该类型时，隐私计算函数返回的也会变为该类型（暂时只支持psi)

**结构**

具体存储数据的文件是按照换行符'\n'作为数据的分隔符，一行就是对应一条数据，每条数据默认是字符串类型：

```lua
一条数据只有一列时，结构如下：
a
b

一条数据只有多列时，结构如下：
a,b,c
d,e,f

```

**方法**

```lua
#获取数据的条数
bds:count()   

#将文件中的数据全部加载出来返回，返回类型是字符串数组
#如果数据集较大可能会造成内存不够而OOM
bds:loadAll()   

#获取用于读取数据的迭代器，一次读取一行
it=bds:iterator()   

#迭代器方法
it:hasNext()  #返回是否还有元素的布尔值
it:next()  #获取下一条数据，返回字符串

```

**示例**

```lua
datacall=require("data_call")
bds=datacall.callWithBigDataset({sql="select id from defaultTable"})

allData=bds:loadAll()
assert(#allData==bds:count())

it=bds:iterator()
while(it:hasNext())
do
   print(it:next())
end

```

### 使用方式

当前仅支持psi调用

在子模型中将获取数据的方式改为调用datacall库的callWithBigDataset，然后直接返回该函数的调用结果

然后在主模型中调用支持的隐私计算函数，调用完返回的结果会改为BigDataset类型

完整示例

发起方子模型submodel-001

```lua
datacall=require("data_call")

function main()
	return datacall.callWithBigDataset({sql="select id from defaultTable"})
end
```

参与方子模型submodel-002

```lua
datacall=require("data_call")

function main()
	return datacall.callWithBigDataset({sql="select id from defaultTable"})
end
```

主模型

```lua
fccall=require("fc_call")

function main()
	p1={
		modelName="submodel-001",
		method="main",
		args={}
	}
	p2={
		modelName="submodel-002",
		method="main",
		args={}
	}
