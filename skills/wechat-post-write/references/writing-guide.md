# 写作风格指南

专业但不严肃的语气，是公众号文章的核心竞争力。

## 语气原则

### 专业但不严肃

**专业**：
- 技术描述准确无误
- 数据引用有据可查
- 术语使用恰当
- 逻辑论证严密

**不严肃**：
- 避免官话套话
- 去掉过度修饰
- 句子有节奏感
- 可以有适当的幽默感

### 语气对比

| 过于严肃 | 专业但不严肃 |
|----------|-------------|
| "本项目是一个优秀的开源工具" | "这个项目太顶了" |
| "该项目在GitHub上获得了大量关注" | "开源没几天就3.5万Star了" |
| "使用该工具可以提高工作效率" | "8小时就能从零开发一个完整项目" |
| "我们建议用户考虑使用此方案" | "可以直接冲了" |

## 标题风格库

### 1. 数据驱动型
- 格式：`[具体数字] + [动作/效果]`
- 示例：`3.5万Star`、`8小时夺冠`、`80.9%碾压级表现`
- 要点：数字要具体，要体现冲击力

### 2. 悬念设置型
- 格式：`[反差/悬念词] + [核心亮点]`
- 示例：`史诗级泄露`、`太顶了`、`杀疯了`
- 要点：用强烈的情感词制造点击欲

### 3. 问题引导型
- 格式：`[提问] + [答案/方向]`
- 示例：`为什么它能夺冠？`、`凭什么这么火？`
- 要点：问题要戳中读者痛点

### 4. 对比反差型
- 格式：`[A] + [对比词] + [B]`
- 示例：`开源平替`、`正面硬刚`
- 要点：找到真正的差异化价值

### 5. 命令式/行动型
- 格式：`[动词] + [动作]`
- 示例：`冲了`、`赶紧用起来`
- 要点：简短有力，号召行动

## 开头写法（前200字法则）

读者只看前200字决定是否继续。开头必须：

1. **抓住注意力**
   - 用数据
   - 用场景
   - 用反差

2. **预告价值**
   - 告诉读者能获得什么

3. **自然过渡**
   - 引出正文内容

### 开头示例

| 类型 | 示例 |
|------|------|
| 数据型 | "开源没几天就 **3.5万 Star** 了" |
| 场景型 | "在8小时的黑客松比赛中，他们..." |
| 悬念型 | "这个项目凭什么能夺冠？" |
| 反差型 | "一个开源工具，竟然碾压商业产品" |

## 段落节奏

### 原则

- 长句 + 短句交替
- 每段控制在100字以内
- 重点信息单独成段
- 适当留白

### 好例子

```
普通写法：
Claude Sonnet 5的性能非常强大，它在多个基准测试中都取得了优异的成绩，尤其是在SWE-Bench上的得分超过了80.9%，远超当前市面上所有的编程大模型，这说明它的编程能力已经达到了一个新的高度。

优化后（节奏感）：
Claude Sonnet 5，碾压级表现。

在SWE-Bench上，它的得分超过 **80.9%**。

此前最高分为74.4%。

一句话：它不只是在写代码，而是在交付代码。
```

## 过渡技巧

### 项目间过渡

| 场景 | 过渡句 |
|------|--------|
| 递进 | "不仅如此，XX还..." |
| 转折 | "然而，XX的局限在于..." |
| 并列 | "同样值得关注的是..." |
| 因果 | "正因为如此，XX成为..." |
| 对比 | "如果说XX做对了A，那么XX选择了B" |

## 结尾收束

### 有力量但不拖沓

| 类型 | 示例 |
|------|------|
| 展望型 | "XX时代，正在到来" |
| 行动型 | "冲就完了" |
| 提问型 | "你准备好了吗？" |
| 金句型 | "这，将是一个彻底的范式升级" |

### 避免
- 不要再重复文章内容
- 不要过度升华
- 不要拖沓冗长

## 常见问题

### Q: 什么时候用"太顶了"这类网络用语？
A: 适度使用可以增加亲和力，但不要过度。技术描述部分保持专业。

### Q: 引用X/Reddit链接是必须的吗？
A: 选择性引用。关键信息（数据、争议、创始人言论）需要标注来源；常识性信息不必引用。

### Q: 段落可以多短？
A: 最短可以是1-2个短句，如" **80.9%**。此前最高分为74.4%。" 这种节奏感很强的断句。
